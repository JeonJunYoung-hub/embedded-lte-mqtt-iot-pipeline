# Nordic nRF9151 Firmware & Embedded System

## Device Overview

This custom embedded system collects environmental data including PM1.0, PM2.5, PM4.0, PM10, CO2, temperature, humidity, VOC, and NOx.

The system dynamically adjusts its sampling interval:

* Default: every 20 minutes
* PM2.5 > 12.1 → every 10 minutes
* PM2.5 > 35.5 → every 5 minutes

---

## System Workflow

1. Device powers on
2. GNSS retrieves satellite data
3. LTE connection is established
4. Sensor data collection begins
5. Data is transmitted via MQTT

LED indicators:

* First blink: GNSS active
* Second blink: LTE connection

---

## Firmware Architecture

### Core Components

* GNSS (location acquisition)
* LTE modem (communication)
* I2C sensor interface
* MQTT client

---

## Sensor Integration

* Sensors connected via I2C (SEN66, BMP581)
* Device Tree used for configuration
* Sensor data sampled multiple times and averaged

---

## Key Functionalities

### GNSS + LTE Workflow

* GNSS activated first to retrieve location
* LTE connection initialized afterward
* Requires multiple satellites for fix
* Timeout handling with retry mechanism

---

### LTE Communication

* LTE modem initialized using `nrf_modem_lib_init()`
* Connection handled via async handler
* Network registration monitored before transmission

---

### Power Optimization (PSM)

* Power Saving Mode configured
* LTE disabled during idle state
* Enabled only during transmission

---

### Watchdog System

* Timeout-based reset mechanism (~15 min)
* Prevents system freeze
* Regular feeding required

---

### RTC-Based Scheduling

* Custom RTC built using counter
* Periodic execution (5 / 10 / 20 min)
* Alarm-based scheduling system

---

### Multithreading

* Sensor measurement and data transmission separated
* Prevents blocking delays
* Improves system responsiveness

---

## Data Processing

* Sensor values sampled multiple times
* Averaged for stability
* Converted into JSON format before transmission

---

## Summary

This firmware enables:

* Reliable embedded sensing
